### Reliability defects

The designated release candidate has not completed manual audit, so the absence of recorded defects is not evidence of reliability. Populate the defect log in `docs/V1.0.9-FEATURE-AUDIT.md` before changing this decision.

### Safety and privacy

Implemented controls include local-only training/model storage, no raw-frame saving in the training path, explicit camera state, deterministic contact release, configuration/termination release calls, binding validation, unknown-capability denial at the Phase 0 boundary, and an explicit No Gesture outcome.

Pending evidence includes real mouse-button release across every interruption, camera resource release, permission grant/revoke recovery, invalid target errors, shell-command containment, configuration deletion/recovery, and excluded-feature non-interference.

### Architecture

The approved 1.0.9 direction is native Swift/macOS first with Apple Vision feeding a detector-neutral canonical 21-joint boundary and local Core ML for eligible static personalization. The typed core is presently a tested seam and mock slice, not yet the sole production dispatch route. That debt is accepted only for an alpha foundation release and must remain visible.

### User feedback

Pending. Record what confused users about Camera state, hand labels, readiness, model errors, click lifecycle, Actions performance, and recovery.

### Scope changes and explicit cuts

The release prioritizes Phase 0 closeout, training clarity, camera/window lifecycle, regression evidence, and Actions performance. It does not ship MediaPipe, model fusion, a general static model, replay/evaluation reports, depth/26-anchor tracking, cloud training, or learned mouse release.

Voice-command reliability, PDF understanding, image understanding, and the 3D model viewer are excluded from the v1.0.9 pass gate but must not crash or degrade included behavior.
