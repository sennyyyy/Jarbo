# ALPHA Jarbo 1.0.9 Feature Audit

## Purpose

This document is the source of truth for what works well, what works with limitations, what does not work, and what is intentionally excluded from the ALPHA Jarbo 1.0.9 closeout.

Do not mark a feature as verified from source inspection or a previous release note alone. A verified result requires a completed test on the v1.0.9 release candidate with the environment and evidence recorded below.

Current decision: **HOLD**. Commit `13c7522` passes clean-checkout automation. GitHub CI is green on later evidence commits `fc1b9b8` and `47ec117`, but those commits are not tree-equivalent to the designated candidate, so candidate-specific CI remains pending. This audit also has no complete hardware run, performance trace, or soak evidence yet.

## Status definitions

| Status | Meaning |
|---|---|
| Not tested | No v1.0.9 release-candidate test has been recorded. |
| Pass | Works consistently and meets its acceptance criteria. |
| Pass with limitations | Core behavior works, but a documented limitation remains. |
| Fail | Does not meet its acceptance criteria or produces incorrect behavior. |
| Blocked | Cannot be tested because a required permission, account, device, app, or environment is unavailable. |
| Excluded | Not required for the v1.0.9 release gate, but must fail safely and must not degrade unrelated features. |
