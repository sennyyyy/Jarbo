# ALPHA Jarbo 1.0.7

## Working

- Added a detector-neutral 21-joint hand-landmark layer so tracking, overlays, gesture rules, and training no longer depend directly on Apple Vision joint types.
- Apple Vision remains the production detector and now feeds canonical Jarbo hand frames with handedness, confidence, detector source, and an optional depth slot for future backends.
- Added a real personalized Core ML classifier for static gestures using normalized landmark features.
- Added `No gesture / other` as a required rejection class to reduce unintended activations.
- Each personalized class still has its own ten-sample requirement. A model can be built after completing No Gesture and at least two static gesture classes; incomplete additional classes are safely ignored until complete.
- Core ML training runs away from the HUD thread, uses the Mac's available compute units, persists under Application Support, and reloads on future launches.
- Training samples now retain detector source, selected hand, and capture time while storing no camera images.
- Personalized predictions use explicit confidence thresholds and fall back to existing templates, deterministic rules, and bundled priors when uncertain.
- Dedicated pinch begin/hold/release logic remains ahead of learned classification so mouse button safety does not depend on a probabilistic model.
- Added in-app model readiness and training status text plus personalized-pipeline documentation.

## Known limitations

- MediaPipe is not bundled in this release. The new detector boundary is ready for an adapter, but its macOS packaging, latency, signing footprint, and accuracy still need an A/B benchmark against Apple Vision.
- The first Core ML model handles static poses only. Orientation samples retain a different coordinate transform, and dynamic gestures require a temporal model.
- A useful model requires deliberate variation in hand angle, distance, and lighting across all ten samples. Ten nearly identical frames can overfit.
- The rejection class only represents the examples captured by the user; collecting varied No Gesture samples improves false-activation resistance.
- Core ML improves gesture interpretation but cannot reconstruct landmarks hidden by severe occlusion or motion blur.
- Pinch accuracy still depends on Apple Vision's monocular fingertip positions until a second detector is integrated and benchmarked.
- Mac control still requires Accessibility and, for desktop switching, System Events Automation permission.
- The downloadable build remains x86_64, ad-hoc signed, and not notarized.
