# ALPHA Jarbo 1.0.4

## Working

- Pointer sensitivity now defaults to 0.5× while retaining the 0.15×–1.2× adjustment range.
- Personal Training now supports up to eight samples for any static built-in or custom gesture. This allows users to train Jarbo on their own fist, pinch, pointing style, and Custom A/B/C poses.
- The local personalized classifier normalizes each sample for wrist position, palm size, and hand rotation, ranks multiple samples per gesture, and rejects ambiguous matches.
- Personalized matches run before generic static-pose rules, allowing trained fist and pinch samples to override one-size-fits-all geometry.
- Generic pinch recognition now requires fingertip reach and evidence of an actual pinch pose, reducing the common closed-fist-as-pinch error.
- Desktop switching now uses macOS System Events to send Control–Left/Right Arrow and automatically falls back to direct HID events if System Events fails.
- The HUD distinguishes a requested System Events action, a confirmed action, and a HID fallback.
- Play/pause, next-track, and previous-track actions now target Spotify by default.
- GitHub releases are now titled **ALPHA Jarbo _version_**.
- Existing v1.0.3 settings migrate to schema 4 and the requested 0.5× sensitivity automatically.

## Known limitations

- The personalized classifier is a small on-device landmark model, not GPT. GPT-based video control would add network latency and is not suited to reliable frame-by-frame mouse input.
- Train 3–5 natural variations of both Fist and Pinch for the best separation. A single sample may be too sensitive to camera angle.
- System Events may request Automation permission the first time desktop switching is used. Jarbo also needs Accessibility permission and must show **Control Ready**.
- macOS must have at least two Spaces, with Control–Left Arrow and Control–Right Arrow enabled in Mission Control keyboard shortcuts.
- Webcam quality still depends on lighting, full hand visibility, and limited finger occlusion.
- Pointer control currently targets the primary display.
- The downloadable build remains x86_64, ad-hoc signed, and not notarized. Apple Silicon requires Rosetta 2.
