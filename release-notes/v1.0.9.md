# ALPHA Jarbo 1.0.9

> Prerelease status: clean-checkout automation passes, but release-candidate hardware, CI, performance, and soak evidence are not yet complete. The Phase 0 decision remains **HOLD** until the audit is signed.

## Working in the repository candidate

- Added typed `JarboEvent`, `JarboIntent`, `Capability`, `ActionRequest`, `ActionResult`, and `AuditEvent` contracts plus resolver, policy, executor, cancellation, and control-release protocols.
- Added a side-effect-free mock gesture-to-intent-to-policy-to-executor vertical slice and foundational tests for verified execution, unknown-capability denial, No Gesture suppression, cancellation, and tracking-loss release.
- Added exact personalized-model readiness logic and tests: ten No Gesture samples and two eligible static classes with ten samples each are mandatory; dynamic, orientation, invalid, and incomplete extra classes cannot enable the static model incorrectly.
- Reworked Personal Training with exact per-class counts, varied-capture prompts, near-duplicate warnings, clearer No Gesture examples, and typed failure feedback that requires a fresh complete 21-joint pose.
- Added persistent model-build diagnostics, local model metadata, and confirmation-backed delete/rebuild controls. Personalized training remains local and stores normalized landmarks rather than camera images.
- Added value validation for URLs and other value-bearing action bindings plus settings/default/migration test coverage.
- Added an explicit Camera On/Off state model and menu control, a distinct template Jarbo menu icon with accessible text, persisted camera intent with a safe new-profile default of Off, and lifecycle cleanup paths for tracking/control state.
- Preserved Jarbo as a regular Dock and Command-Tab application with a normal-level window, predictable close/hide/background behavior, and passive HUD updates that need not steal focus.
- Reduced configuration overhead with cached gesture catalogs, debounced off-main settings persistence, throttled/paused expensive work while Actions is open, bounded training capture, and performance signposts.
- Added dynamic Xcode/SDK environment preflight, a single `./verify.sh` test/build/bundle command, universal arm64/x86_64 verification, and a macOS CI workflow.
- Added product, architecture, risk, smoke-test, feature-audit, and Phase 0 review documents that separate implementation evidence from release verification.

## Verification still required

- The GitHub macOS workflow must be green for that commit.
- Camera On/Off, permission recovery, background behavior, focus/full-screen coexistence, left/right identity, and ten-cycle resource release must be tested on supported hardware.
- Pointer movement and left/right/middle hold-to-click must pass real-device begin/hold/release tests, including interruption by tracking loss, Actions, Camera Off, and quit.
- Every supported action and Actions-editor lifecycle must complete the v1.0.9 audit with understandable failure reporting.
- Personalized build, failed rebuild, delete, rebuild, persistence, and relaunch must be rehearsed with varied samples.
- The Actions editor must meet the 500 ms/100 ms responsiveness targets and pass CPU, memory, ten-cycle, and 30-minute soak gates.

## Known limitations

- This is an alpha development build. It is ad-hoc signed, not Developer ID signed or notarized.
- Apple Vision remains the only bundled hand detector. MediaPipe, depth/LiDAR input, and visionOS anchors are deferred until measured comparisons justify them.
- The personalized Core ML model handles eligible static poses only. Dynamic and orientation samples remain separate and do not count toward static readiness.
- Recognition accuracy, false-activation rate, and landmark-to-result latency do not yet have replayable v1.0.9 benchmark evidence; roadmap targets are not current performance claims.
- Webcam tracking can still degrade with blur, occlusion, low light, incomplete hand visibility, and hand crossing.
- Pointer control is primarily designed for the main display; full multi-display behavior remains unverified.
- Desktop switching requires multiple Spaces, enabled Control–Left/Right Mission Control shortcuts, and Accessibility permission.
- Spotify is the default media target and must be available for the default media controls.
- The typed Phase 0 pipeline is a foundation and mock slice; the entire legacy production gesture route has not yet been migrated behind that policy/executor boundary.
- Voice-command reliability, PDF reading/understanding, image reading/understanding, and the 3D model viewer are excluded from the v1.0.9 gate. They must remain non-blocking and fail safely.
- Shell commands are a high-risk, explicitly user-saved action; 1.0.9 is not a sandboxed third-party skills platform.

## Release decision

**HOLD** until `docs/V1.0.9-PHASE-0-REVIEW.md` records a designated commit, clean-checkout verification, green CI, completed feature/smoke audit, resolved critical/high defects, performance results, safety releases, and reviewer approval.
