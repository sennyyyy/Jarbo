# Jarbo 1.0.1

## Working

- Native macOS menu-bar HUD that continues running in the background.
- On-device Vision hand tracking for up to two hands with corrected selfie mirroring, landscape axes, and left/right labels.
- Green per-hand boxes, skeleton landmarks, gesture names, and live action labels in the camera HUD.
- Left-hand pointer preset with adaptive smoothing, deadzone filtering, a translucent circular cursor, pinch click, and middle-pinch right-click.
- Configurable gestures for Spaces, Mission Control, App Exposé, desktop display, media, volume, URLs, applications, files, web searches, notes, speech, image generation, and explicit shell commands.
- Arc Reactor, Midnight, Crimson, and Matrix themes; draggable HUD cards; camera, system, notes, music, PDF/image, map, audio, and 3D viewer panels.
- Camera-startup crash fixed; the tested build remains open after launch.

## Known limitations

- The current downloadable build is x86_64 and uses Rosetta 2 on Apple Silicon. A standard native arm64 build requires a matching full Xcode toolchain.
- The app is ad-hoc signed and not notarized, so first launch may require right-clicking the app and choosing **Open**.
- Hand accuracy still depends on lighting, an unobstructed hand, and keeping the palm reasonably visible.
- Pointer mapping currently targets the primary display; multi-monitor selection is not implemented yet.
- Voice control requires separate Microphone and Speech Recognition permission.
- Image generation requires the user to supply an OpenAI API key in the app.
- Experimental autonomous screenshot-driven computer control is not enabled; automation runs only from explicit user-configured actions.
