# Jarbo 1.0.3

## Working

- Pinch clicks now have a real lifecycle: thumb/index sends mouse-down, remains held for dragging, and sends mouse-up when the pinch is released. Thumb/middle provides the same hold behavior for right-click.
- Held mouse buttons are automatically released when the pose changes, the hand disappears for three frames, tracking stops, or Jarbo quits.
- Pointer control now defaults to 0.32× sensitivity, with a wider jitter dead-zone, stronger motion smoothing, lower acceleration, and a new 0.15×–1.2× adjustment range.
- Pointer movement emits native drag events while a pinch is held.
- Desktop switching now sends an explicit Control-key-down, arrow-down/up, Control-key-up sequence through the HID event tap.
- Removed the default open-palm Mission Control binding that competed with open-palm swipes. Mission Control now defaults to the peace sign.
- Reduced the swipe distance threshold and retained vertical-motion rejection and cooldown behavior.
- Added point left, point right, point up, point down, thumbs down, and three-finger gesture detection.
- Added default mappings for directional media control, volume down, and App Exposé.
- Added three trainable custom pose slots. Users can capture Custom A, B, or C from either visible hand and bind each pose to any supported action.
- Added test buttons for left click and both desktop directions so Accessibility/keyboard output can be verified separately from camera recognition.
- Existing v1.0.2 configurations migrate to the lower sensitivity and non-conflicting v1.0.3 presets automatically.

## Known limitations

- Accessibility must show **Control Ready** in Jarbo before mouse or keyboard events can control macOS. A newly extracted build may need its Accessibility entry toggled or re-added.
- macOS must have at least two Spaces, and **System Settings → Keyboard → Keyboard Shortcuts → Mission Control** must keep the Control–Left Arrow and Control–Right Arrow shortcuts enabled.
- Custom pose matching is on-device template matching rather than model retraining. Capture a distinct pose in steady lighting and avoid making it too similar to a built-in pose.
- Webcam gesture quality still depends on lighting, hand visibility, and finger occlusion.
- Pointer control currently targets the primary display.
- The downloadable build is x86_64, ad-hoc signed, and not notarized. Apple Silicon Macs require Rosetta 2, and first launch may require right-clicking the app and choosing **Open**.
