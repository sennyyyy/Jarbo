# ALPHA Jarbo 1.0.8

## Working

- Changed Jarbo from a background-only agent into a regular macOS application with a Dock icon and Command-Tab presence while preserving menu-bar access.
- Expanded the menu-bar menu with Show/Hide HUD, Configure Controls, Pause/Resume Hand Controls, personalized Core ML status, and Quit.
- Reopening Jarbo from the Dock or Command-Tab restores the HUD when its window is hidden.
- Debounced configuration persistence so typing, sliders, toggles, and binding edits no longer rewrite the entire configuration file for every individual change.
- Capped hand-overlay publication at 12 FPS while retaining the camera/control pipeline at up to 30 FPS.
- Paused overlay redraws and expensive gesture classification while the control editor is open. Landmark collection and dynamic-motion history remain active for training.
- Configuration mode releases held mouse buttons and disables gesture actions behind the editor.
- Standardized camera processing at 1280×720 and a maximum of 30 FPS when supported instead of requesting an unrestricted high-resolution stream.
- Reuses the Apple Vision hand-pose request and contains per-frame allocations in an autorelease pool.
- Replaced the Intel-only executable with a universal arm64/x86_64 build. Apple-silicon Macs now run Jarbo natively instead of through Rosetta.
- Caps personalized Core ML inference at 15 Hz per hand and reuses the most recent result between predictions.
- Fixed pose learning to capture from a fresh, stable detector snapshot rather than requiring every joint in the latest throttled UI frame.
- Retains briefly low-confidence landmarks for capture stability, requires all 21 canonical joints, and expires snapshots after 0.35 seconds without the selected hand.
- Added live landmark counts plus explicit Saved/Failed training feedback in the controls panel.
- Fixed ten-sample pruning so identical or near-identical sample hashes cannot remove multiple training examples accidentally.
- Debounced changes are flushed during normal app termination.

## Known limitations

- Personalized static Core ML training still requires ten No Gesture samples plus ten samples for at least two static gesture classes.
- Training quality depends on collecting meaningful variations rather than ten captures of an entirely unchanged pose.
- The 12 FPS overlay is intentionally slower than the control pipeline; pointer and pinch processing continue at the camera rate.
- Opening the control editor temporarily suspends gesture actions. This is intentional so editing cannot trigger clicks or system commands.
- MediaPipe remains prepared at the detector boundary but is not bundled yet.
- A monocular webcam can still lose fingertips during severe occlusion, blur, or poor lighting.
- Mac control requires Accessibility and desktop switching can also require System Events Automation permission.
- The downloadable build is universal and ad-hoc signed, but not yet Developer ID signed or notarized.
