# Jarbo 1.0.2

## Working

- Replaced full-camera absolute pointer mapping with relative clutch-style control, so small hand movements cover the screen without stretching the hand or arm to camera edges.
- Added adaptive acceleration, jitter deadzones, landmark-jump rejection, pointer sensitivity control, and a longer loss timeout for steadier cursor movement.
- Mouse movement, left click, right click, and keyboard events now use a HID event source with explicit Accessibility preflight checks.
- Added a prominent control-permission status, an **Enable Control** button, blocked-action messages, and live output confirmation such as `LEFT CLICK SENT`.
- Pinches use palm-width normalization, hysteresis, two-frame confirmation, and nearest-finger selection to reduce missed or duplicate clicks.
- Swipes use a shorter motion window, lower practical distance threshold, vertical-motion rejection, cooldown, and edge-triggered dispatch.
- Finger extension detection is rotation-independent instead of requiring vertically aligned fingers.
- Fixed thumbs-up being hidden behind fist detection.
- Legacy configurations automatically restore left pinch clicks and right open-hand swipe presets for desktop switching; the Actions editor also has a manual essential-preset restore button.
- The camera overlay uses a 16:9 geometry to reduce landmark stretching.
- App signing now includes a stable designated requirement for more consistent macOS Accessibility identity across rebuilds.

## Known limitations

- Accessibility must be enabled for the newly extracted Jarbo app in **System Settings → Privacy & Security → Accessibility** before it can move, click, or send desktop-switching keys.
- The current downloadable build remains x86_64 and requires Rosetta 2 on Apple Silicon.
- The app is ad-hoc signed and not Apple-notarized; first launch may require right-clicking and choosing **Open**.
- Gesture accuracy still depends on lighting, camera visibility, and avoiding heavy finger occlusion.
- Relative pointer control currently targets the primary display only.
- macOS must have more than one Space configured for desktop swipe actions to produce a visible change.
